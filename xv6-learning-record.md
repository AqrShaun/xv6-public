# xv6 学习记录
## Makefile
### 核心作用 
核心作用是自动化编译、链接、生成镜像文件，以及提供运行 / 调试 / 清理等辅助命令
### 总结
1.这份 Makefile 是 xv6 编译 / 运行的核心，自动完成从源码到可运行镜像的全流程：检测编译工具→编译内核 / 用户程序→生成磁盘镜像→提供运行 / 调试 / 清理命令；  
2.核心产物是 xv6.img（内核镜像）和 fs.img（文件系统镜像），通过 make qemu 即可运行 xv6；  
3.关键参数（如 -m32、-Ttext 0x7C00）是为了适配 xv6 的 32 位 x86 内核特性，保证能在 QEMU 中正确运行。  

- make：编译生成 xv6.img 和 fs.img；
- make qemu：运行 xv6；
- make clean：清理编译文件；
- make qemu-gdb：调试 xv6。

## 整体架构
宏内核架构，分为 用户态 和 内核态，应用程序通过系统调用进入内核，由内核统一管理进程、内存、文件、设备、中断。
### 两大空间：用户态 & 内核态
#### 用户态
- 运行普通程序：ls、cat、sh、echo
- 权限低，不能直接访问硬件
- 想做事必须通过 系统调用 进入内核
#### 内核态
- 运行 xv6 内核代码
- 权限最高，可以操作所有硬件、内存、进程
- 所有资源都由内核统一管理
